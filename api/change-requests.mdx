## Overview

The Change Requests API allows you to create, manage, review, and merge changes to wiki documentation. Change requests support a collaborative workflow with draft, review, and approval stages.

## Get My Change Requests

Retrieve all change requests created by the authenticated user.

```
POST /api/method/wiki.api.change_requests.get_my_change_requests
```

### Request

No parameters required.

### Response

<ResponseField name="message" type="array">
  Array of change request objects
  
  <Expandable title="Change Request Object">
    <ResponseField name="name" type="string">
      Unique identifier of the change request
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Title of the change request
    </ResponseField>
    
    <ResponseField name="wiki_space" type="string">
      ID of the wiki space
    </ResponseField>
    
    <ResponseField name="wiki_space_name" type="string">
      Display name of the wiki space
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Status: `Draft`, `In Review`, `Approved`, `Changes Requested`, `Merged`, or `Archived`
    </ResponseField>
    
    <ResponseField name="modified" type="datetime">
      Last modified timestamp
    </ResponseField>
    
    <ResponseField name="archived_at" type="datetime">
      Timestamp when archived (if applicable)
    </ResponseField>
    
    <ResponseField name="merged_at" type="datetime">
      Timestamp when merged (if applicable)
    </ResponseField>
    
    <ResponseField name="change_count" type="integer">
      Number of changes in the request
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -X POST https://your-site.com/api/method/wiki.api.change_requests.get_my_change_requests \
  -H "Authorization: token api_key:api_secret"
```

**Response:**
```json
{
  "message": [
    {
      "name": "CR-2025-0001",
      "title": "Update API documentation",
      "wiki_space": "WS-001",
      "wiki_space_name": "Developer Docs",
      "status": "Draft",
      "modified": "2025-01-15 10:30:00",
      "archived_at": null,
      "merged_at": null,
      "change_count": 5
    }
  ]
}
```

## Create CR Page

Create a new page in a change request.

```
POST /api/method/wiki.frappe_wiki.doctype.wiki_change_request.wiki_change_request.create_cr_page
```

### Parameters

<ParamField path="name" type="string" required>
  Change request identifier
</ParamField>

<ParamField path="parent_key" type="string" required>
  Parent document key
</ParamField>

<ParamField path="title" type="string" required>
  Page title
</ParamField>

<ParamField path="slug" type="string">
  URL slug (auto-generated if not provided)
</ParamField>

<ParamField path="is_group" type="integer" default="0">
  1 for group/folder, 0 for page
</ParamField>

<ParamField path="is_published" type="integer" default="1">
  Publication status
</ParamField>

<ParamField path="content" type="string">
  Markdown content
</ParamField>

<ParamField path="order_index" type="integer">
  Sort order (auto-calculated if not provided)
</ParamField>

<ParamField path="is_external_link" type="integer" default="0">
  1 for external link
</ParamField>

<ParamField path="external_url" type="string">
  External URL if is_external_link is 1
</ParamField>

### Response

Returns the `doc_key` of the newly created page.

## Update CR Page

Update a page in a change request.

```

```

### Parameters

<ParamField path="name" type="string" required>
  Change request identifier
</ParamField>

<ParamField path="doc_key" type="string" required>
  Document key to update
</ParamField>

<ParamField path="fields" type="object" required>
  Object containing fields to update (title, slug, content, is_published, etc.)
</ParamField>

### Example

```bash
curl -X POST https://your-site.com/api/method/wiki.frappe_wiki.doctype.wiki_change_request.wiki_change_request.update_cr_page \
  -H "Authorization: token api_key:api_secret" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "CR-2025-0001",
    "doc_key": "abc123",
    "fields": {
      "title": "Updated Title",
      "content": "# New Content\n\nUpdated documentation."
    }
  }'
```
